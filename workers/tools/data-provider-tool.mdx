## Overview

The Data Providers tool gives your Utari workers access to specialized data from major platforms including LinkedIn, Yahoo Finance, Amazon, Zillow, and Twitter. This powerful capability enables workers to gather market data, analyze trends, research companies, track real estate, and monitor social media—all through simple conversational requests.

## Using Data Providers

### Basic Data Retrieval

<Steps>
  <Step title="Request Data">
    Ask your worker to retrieve specific data:

    ```
        Get the current stock price for Apple (AAPL) from Yahoo Finance
    ```
  </Step>
  <Step title="Worker Executes Call">
    Your worker:

    - Identifies the appropriate data provider (Yahoo Finance)
    - Selects the correct endpoint (stock quote)
    - Executes the API call with parameters (symbol: AAPL)
    - Retrieves the data
  </Step>
  <Step title="Receive Structured Data">
    Data is returned in a structured format:

    ```json
        {
          "symbol": "AAPL",
          "price": 185.92,
          "change": +2.15,
          "change_percent": +1.17%,
          "volume": 52847392,
          "market_cap": "2.89T"
        }
    ```
  </Step>
  <Step title="Use Data">
    The data can be:

    - Displayed in responses
    - Saved to documents
    - Used in analysis
    - Tracked over time
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Data provider call fails or returns error">
    Check:

    - Provider endpoint is spelled correctly
    - Required parameters are provided
    - Parameter values are in correct format
    - Data exists for the requested query
    - Provider service is available
    - Try with simpler query first
  </Accordion>
  <Accordion title="Returned data is incomplete or missing fields">
    Verify:

    - The endpoint supports the data fields you need
    - Data exists for that specific entity
    - You're using the correct endpoint for your needs
    - Review endpoint documentation for available fields
    - Try a different, more comprehensive endpoint
  </Accordion>
  <Accordion title="Can't find the right endpoint">
    Solutions:

    - Ask worker to list all available endpoints for the provider
    - Review endpoint descriptions carefully
    - Try multiple endpoints to compare results
    - Describe your data needs and ask for recommendation
    - Check provider documentation for guidance
  </Accordion>
  <Accordion title="Data seems outdated or inaccurate">
    Consider:

    - Some data updates in real-time, some is delayed
    - Cross-reference with other sources
    - Check timestamp of data retrieval
    - Verify you're querying the right entity (ticker symbol, address, etc.)
    - Some providers cache data for performance
  </Accordion>
  <Accordion title="Rate limit or quota exceeded">
    Best practices:

    - Space out requests instead of making many at once
    - Use batch endpoints when available
    - Cache frequently accessed data
    - Schedule large data collections during off-peak times
    - Consider if you really need all the data
  </Accordion>
</AccordionGroup>
